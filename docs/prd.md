## 1. 프로젝트 개요

Only-One Room은 개발자의 자취방을 탐험하는 형태의 **인터랙티브 포트폴리오 홈페이지**이다.

기존 포트폴리오가 상단 Navigation Bar와 메뉴 클릭으로 콘텐츠를 이동하는 방식이라면, 이 프로젝트는 **방 안의 오브젝트 자체가 메뉴 역할**을 한다. 사용자는 노트북, 책장, 화이트보드 같은 오브젝트를 클릭하며 개발자의 프로젝트와 기술 스택을 탐색한다.

실제 자취방을 참고하되 UI에 맞도록 **2.5D 스타일로 재해석**하며, 게임처럼 과한 연출보다 개발자의 작업 공간을 둘러보는 차분한 탐험 경험을 목표로 한다.

### 핵심 원칙

- 홈페이지 전체가 **하나의 방**이다.
- **Navigation Bar를 사용하지 않는다.**
- 모든 인터랙션은 **페이지 이동 없이 한 화면**에서 이루어진다.
- 방을 둘러보며 프로젝트를 탐험하는 경험을 제공한다.

## 3. 디자인 컨셉

**키워드**

- 2.5D Illustration
- Modern
- Warm
- Cozy
- Minimal
- Developer Workspace

**방향성**

- 실제 자취방을 참고하지만 UI에 맞도록 재해석한다.
- 게임처럼 과한 연출을 지양하고, 개발자의 작업 공간을 탐험하는 느낌을 목표로 한다.
- 정보 패널은 Glassmorphism 스타일로, 배경 방이 계속 보이도록 한다.

2.5D 스타일의 기준
- Flat Illustration를 목표로 하지 않는다.
- 카메라는 약 15~25도 위에서 방을 내려다보는 시점을 사용한다.
- 가구는 측면과 두께를 표현하여 입체감을 만든다.
- 바닥과 벽은 원근감이 느껴져야 한다.
- 모든 오브젝트는 공간 안에 배치된 느낌을 가져야 한다.
- SVG 아이콘을 나열하는 방식은 지양한다.

## 4. 기능 명세

### 4.1 인트로

홈페이지 최초 접속 시 짧은 인트로를 출력한다.

**연출 흐름**

```
검은 화면 → 현관문이 열림 → 카메라가 방 안으로 이동 → 조명이 켜짐 → Welcome → 메인 화면
```

- 인트로 재생 시간: **2~4초**
- 인트로 종료 후 메인 화면(방 전체)으로 전환된다.

### 4.2 메인 화면

- 방 전체가 메인 화면이다.
- **페이지 이동이 존재하지 않는다.**
- 모든 콘텐츠는 방 안에서 표현한다.

### 4.3 인터랙션

| 이벤트 | 동작 |
|--------|------|
| **Hover** | 오브젝트 살짝 확대 · 그림자 · 밝기 증가 |
| **Click** | 카메라 이동 · 오브젝트 확대 · 정보 패널 출력 |

- 모든 인터랙션에서 **페이지 이동은 하지 않는다.**

### 4.4 카메라

- **CSS Transform** 기반으로 카메라가 이동하는 느낌을 구현한다.
- 예시 흐름: `방 전체 → 노트북 클릭 → 노트북 앞으로 이동 → 프로젝트 출력`

### 4.5 정보 패널

- 스타일: **Glassmorphism**, Rounded Corner, Soft Shadow
- 애니메이션: **Fade Animation**
- 패널이 열려도 **배경 방은 계속 보여야 한다.**

### 4.6 미구현 기능 UX

- 아직 구현되지 않은 기능은 **빈 화면을 출력하지 않는다.**
- 대신 **"Under Development"** 또는 **"Work in Progress"** 형태의 안내 UI를 제공한다.
- 사용자는 현재 개발 중인 기능임을 자연스럽게 이해할 수 있어야 한다.

## 5. 오브젝트별 기능

MVP에서는 아래 4개 오브젝트와 배경 요소(창문)만 사용한다.

| 오브젝트 | 역할 | 클릭 시 동작 |
|----------|------|--------------|
| ① 노트북 | 현재 진행 중인 프로젝트 | 카메라가 노트북 앞으로 이동 → 대표 프로젝트 출력 |
| ② 책장 | 사용 가능한 기술 스택 | 정보 패널에 기술 스택 목록 출력 |
| ③ 화이트보드 | 개발 예정 프로젝트 | 정보 패널에 예정 프로젝트(더미) 출력 |
| ④ 벽 스위치 | Light / Dark Mode 전환 | 방 전체 Theme 변경 |
| 창문(배경) | 현재 Theme 표현 | 클릭 불가, 배경 요소로만 사용 |

### 5.1 노트북 (핵심 인터랙션)

- 가장 중요한 인터랙션이다.
- 클릭 시 카메라가 노트북 앞으로 이동하며 현재 진행 중인 대표 프로젝트를 보여준다.
- 현재는 **Melolist-v3** 하나만 표시한다.
- 향후 프로젝트가 늘어날 경우 모니터 내부에서 좌우로 프로젝트를 넘길 수 있도록 **구조만 고려**한다.
- ⚠️ 이번 구현에서 **슬라이드 기능은 구현하지 않는다.** (확장 대비 구조만 설계)

### 5.2 책장

사용 가능한 기술 스택을 소개한다.

- 예시: Java, Spring Boot, React, Supabase, Docker, Git, AWS 등

### 5.3 화이트보드

앞으로 개발 예정인 프로젝트를 소개한다.

- 예시: AI Room Portfolio, Mobile Application, Spring AI, 기타 아이디어
- 현재 구현에서는 **더미 데이터만 사용**한다.

### 5.4 벽 스위치

- 클릭 시 방 전체의 Theme를 변경한다. (`Light Mode ↔ Dark Mode`)
- Theme 변경 시 방의 전체 분위기가 함께 변경되어야 한다.
  - 조명 / 창문 밖 / 블라인드 / 그림자 등

## 6. UI 구조

```
┌─────────────────────────────────────────────┐
│                  Room (방 전체)               │
│                                               │
│   [창문/배경 - 현재 Theme]                     │
│                                               │
│   ┌────────┐   ┌────────┐                     │
│   │ 화이트  │   │  책장   │                     │
│   │ 보드   │   │         │                     │
│   └────────┘   └────────┘                     │
│                                               │
│        ┌──────────┐        ● 벽 스위치         │
│        │  노트북   │                            │
│        └──────────┘                            │
│                                               │
│   ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐          │
│   │  Information Panel (Glassmorphism)│  ← 오버레이│
│   └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘          │
└─────────────────────────────────────────────┘
     ↑ CameraController가 Transform으로 확대/이동
```

- 단일 화면(SPA) 구조이며 라우팅이 없다.
- 정보 패널은 방 위에 오버레이로 표시되며 배경 방이 비쳐 보인다.

## 7. 컴포넌트 구조

```
App
├── ThemeProvider        // Light / Dark Theme 컨텍스트
├── Intro                // 최초 접속 인트로 연출
├── CameraController     // CSS Transform 기반 카메라 이동
└── Room                 // 방 전체
    ├── RoomObject       // 노트북 / 책장 / 화이트보드 / 벽 스위치 (재사용)
    └── InformationPanel // 선택된 오브젝트 정보 패널
```

| 컴포넌트 | 책임 |
|----------|------|
| `App` | 전체 상태 소유 및 하위 컴포넌트 조립 |
| `Intro` | 인트로 연출 재생 및 종료 처리 |
| `Room` | 방 레이아웃 및 오브젝트 배치 |
| `RoomObject` | 개별 오브젝트의 Hover/Click 인터랙션 (props로 타입 구분) |
| `InformationPanel` | 선택된 오브젝트의 콘텐츠를 Glassmorphism 패널로 출력 |
| `CameraController` | 선택 오브젝트에 맞춰 Transform 값 계산·적용 |
| `ThemeProvider` | Light/Dark Theme 상태 제공 및 전역 스타일 전환 |

| 상태 | 설명 |
|------|------|

| Theme | Light / Dark 현재 테마 |
| Camera | 현재 카메라 위치(선택된 오브젝트 기준 Transform) |
| Selected Object | 현재 선택된 오브젝트 |
| Information Panel | 정보 패널 열림/닫힘 상태 |

## 11. 향후 확장 계획 🚧

아래 기능은 프로젝트의 **최종 목표**이며, **이번 구현에는 포함하지 않는다.**

| 확장 기능 | 설명 |
|-----------|------|
| Supabase 연동 | 프로젝트 데이터 백엔드 연동 |
| 프로젝트 다중 관리 | 여러 프로젝트 데이터 관리 |
| 프로젝트 슬라이드 | 노트북 모니터 내 좌우 프로젝트 전환 |
| Vercel 배포 | 프로덕션 배포 |
| 추가 오브젝트 | 방 내 인터랙션 오브젝트 확장 |
| 사용자 커스터마이징 | 방 구성 사용자 편집 |
| 방 템플릿 선택 | 다양한 방 템플릿 제공 |
| AI 기반 방 생성 | AI로 방 자동 생성 |

> 💡 MVP 단계에서는 위 기능들이 동작하지 않으며, 노출이 필요한 영역은 **"Under Development" / "Work in Progress"** 안내 UI로 대체한다.
