# Only-One Room — Design Doc

> 2026-07-13 UI 개선 작업의 디자인 결정 기록. PRD([prd.md](./prd.md))의 시각 목표(2.5D · Warm · Cozy · Minimal · Glassmorphism)를 전제로 한다.

| 항목 | 내용 |
|------|------|
| 문서 목적 | 디자인 컨셉·토큰·변경 내역을 기록해 이후 작업의 기준으로 삼는다 |
| 적용 범위 | `index.html`, `src/styles.css`, `src/App.tsx`, `src/components/*` |
| 데이터 | `src/data/content.ts`는 변경하지 않음 (콘텐츠는 그대로) |

## 1. 디자인 컨셉

### 시그니처: "패널이 곧 오브젝트"

정보 패널이 클릭한 사물의 재질과 형태를 물려받는다. 내용의 형태가 곧 정보 구조가 되도록 한 것이 이번 개선의 핵심 결정이다.

| 오브젝트 | 패널 스킨 | 형태가 말해주는 것 |
|----------|-----------|-------------------|
| 노트북 | 에디터 창 — 트래픽 라이트, `~/projects/melolist-v3` 경로, `● RUNNING` LED | "지금 실행 중인(진행 중인) 프로젝트" |
| 책장 | 나무 선반 위에 꽂힌 책등 — 카테고리별 선반, 불규칙한 높이, 기울어진 마지막 책 | "쌓아 온 기술이 책처럼 꽂혀 있다" |
| 화이트보드 | 보드 표면 — 마커 손글씨, 마스킹테이프 포스트잇, 하단 마커 트레이 | "아직 스케치 단계인 계획" |

### 두 개의 목소리

- **HUD (방 바깥의 목소리)** — 개발자의 터미널 톤. 모노스페이스, 낮은 채도, 짧은 문장.
  - 힌트: `gnuke@room:~$ 오브젝트를 클릭해 방을 둘러보세요 ▮` (커서 깜빡임)
  - 이름표: 이름 + `// developer · github ↗` 주석 라인
  - 테마: `theme: ☀ day` ↔ `theme: ☾ night`
  - 뒤로가기: `← 방 전체 보기 [esc]`
- **패널 (방 안의 목소리)** — 사물의 재질. 유리(Glassmorphism) 껍데기는 PRD대로 유지하되, 내용물은 위 표의 스킨을 따른다.

## 2. 디자인 토큰

### 팔레트

방 렌더 이미지가 색의 대부분을 담당하므로, UI 팔레트는 이미지에서 추출한 소수의 색만 쓴다.

| 이름 | Light | Dark | 용도 · 유래 |
|------|-------|------|-------------|
| jersey-red | `#d2372e` | `#e04b42` | 액센트. 방 벽의 30번 농구 저지에서 추출 — 핀, 브랜치 배지, 리스트 마커 |
| lamp-amber | `#b97f36` | `#e6b566` | 보조 액센트. 방의 스탠드 조명 — 프롬프트, eyebrow, 모노 강조 |
| walnut-ink | `#3b3226` | `#ece3d5` | 본문 텍스트 |
| parchment glass | `rgba(253,250,242,.74)` | `rgba(26,22,33,.66)` | 패널/HUD 유리 배경 |
| frame | `#e7ddc8→#c3b493` | `#1a1622→#0b0a11` | 레터박스 그라데이션 (디오라마 프레임) |
| board-white | `#fcfbf5` | `#f2efe6` | 화이트보드 표면 |

마커 색(보드 전용): 검정 `#2e3440` · 빨강 `#c2402f` · 파랑 `#33518c` — 방 이미지 속 실제 보드마커 3색.

### 타이포그래피 — 3성부

| 역할 | 서체 | 쓰이는 곳 |
|------|------|-----------|
| 본문/제목 | **Pretendard Variable** (CDN 로드) | 패널 본문, 제목, 이름표 |
| 개발자 보이스 | **JetBrains Mono** | 프롬프트, eyebrow(`// current-project`), 상태 배지, 기술 칩, 단축키 |
| 마커 손글씨 | **Nanum Pen Script** | 화이트보드 패널 전용 (제목·배지) — 다른 곳에 쓰지 않는다 |

이전에는 Pretendard가 선언만 되고 로드되지 않아 맑은 고딕으로 폴백되고 있었다. `index.html`에서 세 서체를 모두 로드한다.

### 모션

- 카메라: `cubic-bezier(0.66, 0, 0.2, 1)` 1s — 기존 유지
- 패널 등장: 18px 상승 + 스케일 0.98→1, 0.5s
- 상시 애니메이션은 3개만: 핀 펄스, 프롬프트 커서, RUNNING LED. 나머지는 전부 인터랙션 반응.
- `prefers-reduced-motion`: 전환 즉시 완료 + 상시 애니메이션 정지

## 3. 변경 내역

### HUD (`App.tsx`, `styles.css`)

- 네 모서리의 동일한 유리 알약 4개 → 각자 다른 역할이 드러나는 터미널 톤 HUD로 교체.
- HUD·패널을 `.viewport` 밖 `.hud` 레이어(화면 기준)로 분리 — 모바일 크롭의 영향을 받지 않는다.
- `Esc` 키로 패널 닫기 추가 (뒤로가기 버튼에 `esc` 키 힌트 표기).

### 정보 패널 (`InformationPanel.tsx`, `styles.css`)

- 공통: eyebrow를 `// comment` 형식의 모노 표기로, 제목은 Pretendard 800. 패널 높이는 내용에 맞게(hug) 변경 — 이전에는 내용이 적어도 화면 전체 높이로 늘어났다.
- 노트북: 상단에 창 타이틀바(`win-bar`) 추가. 하이라이트 마커 `›`, 기술 칩은 모노 토큰, GitHub 버튼은 잉크색으로.
- 책장: 2열 카드 그리드 → 카테고리별 선반 + 책등. 책등 글자색은 배경 밝기(YIQ)로 자동 결정(`spineTextColor`), 높이·폭은 고정 패턴 순환으로 들쭉날쭉하게, 3권 이상인 선반의 마지막 책은 기울어짐.
- 화이트보드: 빈 유리판 + 카드 1장 → 보드 표면 위 마커 손글씨 제목, 포스트잇(짝수 번째는 색과 기울기가 다름), 손으로 그린 타원 배지, 마커 트레이.

### 카메라 (`CameraController.tsx`)

- 패널이 열리면 초점을 화면 중앙(0.5)이 아니라 **패널이 가리지 않는 영역의 중앙**으로 이동: 데스크톱 x≈0.33, 모바일(하단 시트) y≈0.29.
- 이전에는 줌인한 오브젝트가 패널 뒤에 가려졌다 (책장 클릭 시 책장이 안 보이는 문제).

### 레터박스 / 모바일 (`styles.css`)

- 레터박스: 단색 베이지 → 라디얼 그라데이션 + viewport에 그림자·내부 비네트. "디오라마 프레임" 느낌.
- 세로 화면(≤720px, portrait): 방을 `width: min(134dvh, 165vw)`로 cover 크롭 — 좌우가 잘리는 대신 방이 화면을 채운다. 165vw 상한은 4개 핫스팟(x 0.15~0.8)이 모두 시야에 남는 한계값.

### 인트로 (`Intro.tsx`)

- `$ open ~/only-one-room ▮` 부트 라인 추가, 하단 타이틀을 모노 소문자 `only-one-room`으로. 문 열림 연출은 기존 유지.

### 접근성

- `:focus-visible` 전역 아웃라인(jersey-red), 방 오브젝트는 확대 레이어의 붉은 링으로 포커스 표시.
- 패널 `role="dialog"`, 핀·트레이 등 장식 요소 `aria-hidden`.

## 4. 유지보수 가이드

| 하고 싶은 것 | 고치는 곳 |
|--------------|-----------|
| 색·서체·이징 바꾸기 | `styles.css` 상단 `:root` / `:root[data-theme='dark']` 토큰 |
| 프로젝트·기술·예정 프로젝트 내용 | `src/data/content.ts` (책등 색은 `techStack[].color`) |
| 책장 선반 순서 | `InformationPanel.tsx`의 `SHELF_ORDER` |
| 책등 높이·폭 패턴 | `InformationPanel.tsx`의 `BOOK_HEIGHTS` / `BOOK_WIDTHS` |
| 패널 열릴 때 카메라 목표점 | `CameraController.tsx`의 `tx` / `ty` |
| 모바일 크롭 정도 | `styles.css`의 `min(134dvh, 165vw)` |

**주의**: 화이트보드 패널의 마커 색은 테마 토큰이 아니라 고정 hex다 — 보드는 밤에도 흰색이고 마커 색은 물리적으로 변하지 않기 때문. 다크 모드에서 색이 안 바뀐다고 버그가 아니다.
