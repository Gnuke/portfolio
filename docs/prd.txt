# 목적

이 문서는 Claude가 PRD.md를 생성하기 위한 요구사항 문서이다.

PRD는 프로젝트의 최종 방향성을 담되,
현재 구현 범위(MVP)를 명확하게 구분하여 작성한다.

향후 확장 예정인 기능은 PRD에 계획으로만 포함하고,
현재 구현 대상에는 포함하지 않는다.

---

# 프로젝트 정보

프로젝트명

Only-One Room

프로젝트 목적

개발자의 자취방을 탐험하는 형태의 인터랙티브 포트폴리오 홈페이지를 제작한다.

기존 포트폴리오처럼 메뉴를 클릭하는 방식이 아니라,
방 안의 오브젝트를 클릭하여 프로젝트를 탐색하는 경험을 제공한다.

실제 자취방을 참고하여 2.5D 스타일로 재해석한다.

---

# 프로젝트 컨셉

홈페이지 전체가 하나의 방이다.

Navigation Bar는 사용하지 않는다.

방 안의 오브젝트 자체가 메뉴 역할을 수행한다.

모든 인터랙션은 페이지 이동 없이
한 화면에서 자연스럽게 이루어진다.

방을 둘러보며 프로젝트를 탐험하는 경험을 제공한다.

---

# 디자인 컨셉

- 2.5D Illustration
- Modern
- Warm
- Cozy
- Minimal
- Developer Workspace

실제 자취방을 참고하지만
UI에 맞도록 재해석한다.

게임처럼 과한 연출보다
개발자의 작업 공간을 탐험하는 느낌을 목표로 한다.

# 인트로

홈페이지 최초 접속 시
짧은 인트로를 출력한다.

예시

검은 화면

↓

현관문이 열림

↓

카메라가 방 안으로 이동

↓

조명이 켜짐

↓

Welcome

↓

메인 화면

인트로는 2~4초 정도를 목표로 한다.

---

# 메인 화면

방 전체가 메인 화면이다.

페이지 이동은 존재하지 않는다.

모든 콘텐츠는 방 안에서 표현한다.

---

# 오브젝트 구성

현재 MVP에서는 아래 오브젝트만 사용한다.

① 노트북

현재 진행 중인 프로젝트

② 책장

사용 가능한 기술 스택

③ 화이트보드

개발 예정 프로젝트

④ 벽 스위치

Light Mode / Dark Mode 전환

창문은 현재 Theme를 표현하는 배경 요소로만 사용한다.

---

# 노트북

가장 중요한 인터랙션이다.

노트북 클릭 시

카메라가 노트북 앞으로 이동하며

현재 진행 중인 대표 프로젝트를 보여준다.

현재는

Melolist-v3

하나만 표시한다.

향후 프로젝트가 늘어날 경우

모니터 내부에서 좌우로 프로젝트를 넘길 수 있도록 설계한다.

단,

이번 구현에서는

슬라이드 기능은 구현하지 않는다.

향후 확장 가능하도록 구조만 고려한다.

# 카메라

예시

↓

노트북 앞으로 이동

↓

프로젝트 출력

# PRD 작성 요청

위 요구사항을 기반으로

구현 가능한 수준의 PRD.md를 작성한다.

PRD에는

- 프로젝트 개요
- 프로젝트 목표
- 디자인 컨셉
- 기능 명세
- 오브젝트별 기능
- UI 구조
- 컴포넌트 구조
- 상태 관리
- MVP 구현 범위
- 향후 확장 계획

을 포함하여 체계적으로 작성한다.

향후 확장 계획과
현재 구현 범위를 명확하게 구분하여 작성한다.
